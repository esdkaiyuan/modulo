# Animation And Video Modulo Design

## Goal

Add a pure-frontend animation/video modulo workflow to the existing Vue tool. The feature lets users upload GIF animations or browser-decodable videos, extract frames with user-controlled limits, convert each frame to dot-matrix data using the existing modulo settings, and copy or export the generated data in multiple C-friendly formats.

## Current Context

The project is a Vue 3 + Vite single-page app. Most functionality lives in `src/App.vue`.

Existing workflows follow this pattern:

- left sidebar tab selection
- shared dot-matrix configuration in the sidebar
- upload/input section
- generate button
- preview and data output section
- copy/export buttons

The existing static image workflow already converts image pixels to mono or color data. The animation/video workflow should reuse the same scan, encoding, byte-order, threshold, and width/height concepts so generated frames are compatible with the rest of the tool.

## Scope

### In Scope

- Add a new sidebar tab named `动画/视频取模`.
- Support GIF and browser-decodable video files such as MP4 and WebM.
- Use FFmpeg WebAssembly in the browser to normalize media extraction.
- Extract frames using these controls:
  - fixed FPS
  - maximum frame count
  - combined FPS plus maximum frame count limit
- Convert each extracted frame into dot-matrix bytes.
- Provide three output formats:
  - per-frame C arrays
  - complete animation C structure with width, height, frame count, frame delay, and frame pointers/data
  - raw hexadecimal frame data
- Provide media preview, extracted frame thumbnails, and a dot-matrix animation preview.
- Provide copy and export actions for the selected output format.
- Show loading, extraction, and generation progress.
- Guard against heavy jobs by using conservative defaults and user-visible limits.

### Out Of Scope

- Server-side media processing.
- Audio extraction or audio playback.
- Editing, trimming, cropping, filters, subtitles, or timeline tools.
- Persisting uploaded media outside browser memory.
- APNG/WebP animated frame extraction in the first version.
- Pixel-perfect performance for very large videos.

## User Experience

### Navigation

Add a fifth sidebar item:

```text
动画/视频取模
```

The tab opens a dedicated workflow rather than extending the static image tab. Static image conversion and animated media conversion have different loading states, controls, previews, and output size concerns, so a separate tab keeps the UI easier to scan.

### Input Section

The input area accepts click-to-upload and drag-and-drop.

Accepted file guidance:

```text
支持 GIF、MP4、WebM 等浏览器可解码视频
```

When a file is selected:

- store the `File`
- show filename, type, and approximate size
- create an object URL for preview

- do not start extraction until the user clicks generate

### Extraction Controls

Controls shown after upload:

- extraction mode:
  - `按帧率`
  - `按最大帧数`
  - `帧率 + 最大帧数`
- FPS input/range, default `5`, min `1`, max `30`
- max frame count input/range, default `64`, min `1`, max `256`
- frame delay override toggle
- frame delay input, default derived from FPS as `Math.round(1000 / fps)`

The shared sidebar settings continue to control:

- width
- height
- scan mode
- encoding mode
- byte order

### Progress States

The workflow has these user-visible phases:

- idle
- loading FFmpeg core
- writing input file
- extracting frames
- converting frames
- complete
- error

The generate button is disabled while processing. The UI shows phase text and progress percentage when available.

### Result Section

After generation, show:

- original media preview
- extracted frame count and generated byte count

- output tabs:
  - `逐帧数组`
  - `动画结构体`

## Media Extraction Architecture

### FFmpeg Loading

Use FFmpeg WebAssembly as a lazy dependency. It should load only when the user first uses the animation/video workflow, not during initial page load.

Implementation options can use the current FFmpeg browser package APIs available at implementation time, but the app-facing wrapper should stay small:

```js
const ensureFfmpegLoaded = async () => {}
const extractMediaFrames = async (file, options) => {}
```

This wrapper should be the only part that knows FFmpeg command details.

### Frame Extraction

The extractor writes the uploaded file into FFmpeg's in-memory filesystem and emits PNG frames.

For FPS-based extraction, the command should use an FPS video filter equivalent to:

```text
-vf fps=<fps>
```

For maximum frame count, generation should stop reading frames after `maxFrames`. If FFmpeg supports an efficient frame limit for the chosen command, use it; otherwise enforce the limit when collecting output files.

For combined mode, use both FPS extraction and the max frame limit.

### Frame Conversion

Each extracted PNG frame is loaded into an `Image`, drawn into a temporary canvas at the configured width and height, then converted to bytes using the same scan, encoding, byte-order, and threshold rules used for static images.

The conversion logic should be extracted into a reusable helper so static image, video frames, and future workflows do not duplicate byte-packing loops.

## Output Formats

### Per-frame C Arrays

Generate one array per frame:

```c
const uint8_t anim_frame_000[] = {
    0x00, 0x18
};
```

### Complete Animation Structure

Generate a complete copy-ready block:

```c
#include <stdint.h>

#define ANIM_WIDTH 16
#define ANIM_HEIGHT 16
#define ANIM_FRAME_COUNT 12
#define ANIM_FRAME_DELAY_MS 200
#define ANIM_BYTES_PER_FRAME 32

const uint8_t anim_frame_000[] = {
    0x00, 0x18
};

const uint8_t* const anim_frames[ANIM_FRAME_COUNT] = {
    anim_frame_000
};

typedef struct {
    uint16_t width;
    uint16_t height;
    uint16_t frameCount;
    uint16_t frameDelayMs;
    uint16_t bytesPerFrame;
    const uint8_t* const* frames;
} AnimationBitmap;

const AnimationBitmap animation_bitmap = {
    ANIM_WIDTH,
    ANIM_HEIGHT,
    ANIM_FRAME_COUNT,
    ANIM_FRAME_DELAY_MS,
    ANIM_BYTES_PER_FRAME,
    anim_frames
};
```

### Raw Hexadecimal Frame Data

Generate frame-separated hex:

```text
// frame 0
0x00, 0x18

// frame 1
0x00, 0x3C
```

## Error Handling

Show a clear error message when:

- no file has been selected
- FFmpeg fails to load
- the uploaded file cannot be decoded
- no frames are extracted
- the generated output would exceed configured limits
- canvas/image decoding fails

Do not leave stale results visible after a failed generation. Keep the uploaded file and settings so the user can adjust controls and retry.

## Performance And Safety Limits

Defaults:

- FPS: `5`
- max frames: `64`
- maximum selectable frames: `256`

The UI should warn when:

- FPS is above `15`
- max frames is above `128`
- estimated bytes exceed a visible threshold

Byte estimate:

```js
const bytesPerFrame = scanMode.value === 'row'
  ? Math.ceil(width.value / 8) * height.value
  : Math.ceil(height.value / 8) * width.value
const estimatedBytes = bytesPerFrame * mediaMaxFrames.value
```

## Implementation Notes

The current `src/App.vue` is already large. The implementation should keep changes targeted, but the media feature is big enough to justify extracting small helpers under `src/utils/` for:

- byte packing
- media output formatting
- FFmpeg frame extraction

The Vue UI can remain in `App.vue` for this iteration to avoid a broad component refactor, but reusable pure logic should not be buried inside the component.

## Acceptance Criteria

- Users can upload GIF or video files.
- Users can choose FPS and maximum frame limits.

- Users can preview extracted frames and dot-matrix animation.

- Production build succeeds.
